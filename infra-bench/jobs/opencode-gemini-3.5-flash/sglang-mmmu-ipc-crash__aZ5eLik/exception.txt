## The failure

Under concurrent multimodal load the scheduler dies with:

```
Failed to deserialize from CUDA IPC handle (HIP error: invalid device pointer
  ... reconstruct_on_target_device (cuda_ipc_transport_utils.py)
torch.AcceleratorError: HIP error: invalid device pointer
SIGQUIT received. ... It usually means one child failed.
```

The multimodal image features are passed between processes over a **CUDA/HIP IPC
transport**. Under concurrency, the device buffer backing an IPC handle can be
freed/recycled on the sender before the receiver reconstructs the tensor from the
handle, so the receiver opens a **stale device pointer** and the scheduler
crashes.

## What to do

1. Diagnose the crash from the log and the serving configuration.
2. **Fix the server configuration** so the multimodal server survives the MMMU
   test — **without disabling the CUDA IPC multimodal transport** (the fix must
   keep that optimization on, not turn it off).
3. Write your analysis to **`/app/analysis.json`**:

```json
{
  "root_cause": "<what makes the CUDA IPC handle point at freed/recycled device memory under concurrency>",
  "fix": "<the exact env var / setting you changed to fix it>",
  "evidence": "<the log line(s) that identify the failure>"
}
```

Your fix must be reflected in `/app/launch_qwen3.5_27b_sglang.sh` (the grader
launches **that script**).

## How you are graded

The grader does **not** trust your description. It launches **your**
`/app/launch_qwen3.5_27b_sglang.sh`, then:

1. **analysis_present** — `/app/analysis.json` exists and parses.
2. **launch_config_valid** — the running scheduler'"'"'s actual process environment
   shows the CUDA IPC multimodal transport **still enabled** AND the IPC pool
   handle cache **enabled** (read from the live process, not from the file).
3. **server_healthy** — the server reaches `/health` = 200.
4. **mmmu_benchmark_intact** — the MMMU benchmark source matches a build-time
   snapshot (editing `bench_sglang.py` to fake accuracy fails here).
5. **mmmu_accuracy** — the grader runs the MMMU benchmark itself (concurrency
   128) and requires a real overall accuracy (> 0.28).
6. **agent_identified_rootcause** — your `root_cause` names the correct root cause
   and your `fix` names the setting you changed.
